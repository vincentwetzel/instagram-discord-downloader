# AI Agent Instructions

When interacting with this repository as an AI coding assistant (e.g., Gemini Code Assist, GitHub Copilot, etc.), please adhere to the following guidelines:

## 1. Code Style & Standards
- Follow PEP 8 guidelines for Python code.
- Provide complete type hints (`typing` module) for all function arguments and return types.
- Keep functions modular and clearly documented using standard docstrings.
- Prefer code lines around 88 characters where practical to conserve coding-agent context. This is a soft rule; avoid unnatural wrapping when a slightly longer line is clearer.

## 2. Architecture Constraints
- **No blocking the event loop:** the downloader is synchronous and heavily I/O bound. Any calls to it from `discord_bot.py` MUST be wrapped in `asyncio.to_thread` or equivalent.
- **Database Syncing:** Any changes to how posts are tracked must update the SQLite database logic (`download_history.db`) in the downloader history module. Rely on SQL constraints (`INSERT OR IGNORE`) for idempotency.
- **Discord Limits:** Discord messages have a 2000 character limit. Ensure reports generated by the downloader and sent via the bot are properly truncated.
- **Module boundaries:** Keep Discord command handling in `discord_bot.py`, session orchestration in `downloader.session`, saved-post workflow logic in `downloader.downloads`, SQLite operations in `downloader.history`, and report formatting in `downloader.reporting`.

## 3. Libraries
- Discord: `discord.py`
- Instagram: `instaloader`
- Database: Standard library `sqlite3`
