# AI Agent Instructions

When interacting with this repository as an AI coding assistant (e.g., Gemini Code Assist, GitHub Copilot, etc.), please adhere to the following guidelines:

## 1. Code Style & Standards
- **Strict Compliance:** All code modifications and additions MUST follow the rules and constraints defined in `CODING_STANDARDS.md`.
- Follow PEP 8 guidelines for Python code.
- Provide complete type hints (`typing` module) for all function arguments and return types.
- Keep functions modular and clearly documented using standard docstrings.

## 2. Architecture Constraints
- **No blocking the event loop:** `instaloader_downloader.py` is synchronous and heavily I/O bound. Any calls to it from `discord_bot.py` MUST be wrapped in `asyncio.to_thread` or equivalent.
- **Database Syncing:** Any changes to how posts are tracked must update the SQLite database logic (`download_history_<account>.db`) in `downloader.history`. Rely on SQL constraints (`INSERT OR IGNORE`) for idempotency.
- **Account Scope:** The downloader processes one configured Instagram account per run. Users may switch accounts between runs by updating `ig_name` and switching Firefox to that account; keep account-specific databases and download paths keyed by the configured account name.
- **Discord Limits:** Discord messages have a 2000 character limit. Ensure reports generated by the downloader and sent via the bot are properly truncated.
- **Media Fallbacks:** Blob-video resolution in `downloader.downloads` should prefer progressive MP4 streams with audio from metadata sources before falling back to DASH or playback-captured streams. Carousel video deduplication should use stable asset identifiers, not raw CDN URLs alone.

## 3. Libraries
- Discord: `discord.py`
- Instagram/browser automation: `playwright` and `playwright-stealth`
- Database: Standard library `sqlite3`
